Arkloop 提供三层记忆，各自独立运作，也可以组合使用以获得完整覆盖。

## Notebook（笔记本）— 给 AI 留便签

Notebook 是手动笔记系统。你主动写的笔记，每次对话都会注入上下文。无需配置模型，开箱即用。

适合固定信息：个人偏好、项目背景、角色设定、编码规范——任何"永远需要 AI 知道"的内容。

操作：设置 → Notebook → 添加 / 编辑 / 删除 / 搜索笔记。

## Memory（语义记忆）— AI 自主的大脑

Memory 是语义记忆系统。AI 自动整理对话中的重要信息，只在相关时召回。你不需要手动管理条目——AI 决定存储什么、何时检索。

适合动态信息：过去的决策、演变的偏好、随时间浮现的上下文——你不想手动整理但希望 AI 记住的内容。

需要配置：提取模型 + 嵌入模型（可选 Rerank 模型提升检索精度）。后端选项：OpenViking（内置向量存储）或 Nowledge（外部知识图谱）。

操作：设置 → Memory → 启用 → 选后端 → 配置模型。

## Nowledge（知识图谱）— Memory 的另一种后端

Nowledge 是 Memory 的替代后端，用知识图谱替代 OpenViking 的向量存储。提供 Working Memory、召回记忆、线程级历史检索，支持知识图谱的 connections 浏览和 timeline 浏览。

适合已自建 Nowledge 实例的高级用户，需要图结构召回而非扁平向量搜索。

配置：Base URL + API Key + Timeout，支持检测本地实例。

## Notebook vs Memory

| | Notebook | Memory |
|---|---|---|
| 谁管理 | 你手动写 | AI 自动整理 |
| 何时注入 | 每次对话 | 只在相关时 |
| 需要配置 | 不需要 | 提取/嵌入模型 |
| 适合 | 固定信息 | 动态信息 |

## 如何感知记忆在工作

对话中，MemoryActionBlock 面板实时显示记忆操作——写入、搜索、读取、删除。AI 也会间接表现出"记得"之前的内容，无需你再次提醒。

## 印象画像与快照

**印象画像（Impression）** 是 AI 对你的整体画像，跨所有对话累积。可在设置中查看和重建。

**快照（Memories）** 显示当前记忆条目，支持 L1/L2 层级浏览。L1 是精简摘要，L2 提供更丰富的细节。可在设置中查看和重建。
