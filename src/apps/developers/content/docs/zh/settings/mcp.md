## 什么是 MCP

MCP（Model Context Protocol）是一种标准协议，用于将外部工具和数据源连接到你的 Agent。Arkloop 不需要内置所有能力，而是通过 MCP 接入第三方服务——数据库、API、内部工具——Agent 使用它们的方式与内置工具无异。

MCP 服务器连接后，其暴露的所有工具自动对 Agent 可用。

## 添加 MCP 服务器

设置 → MCP → 添加服务器。

每个服务器需要配置：

- **显示名称** — 服务器的可读标签
- **传输类型** — Arkloop 与服务器通信的方式

### HTTP 模式

HTTP 模式适用于网络可达的远程 MCP 服务器。

| 字段 | 说明 |
|------|------|
| URL | 服务器端点（如 `https://mcp.example.com/sse`） |
| Headers | 自定义 HTTP 头（键值对） |
| Bearer Token | 认证令牌，通过 `Authorization` 头发送 |
| Timeout | 请求超时时间（秒） |

### Stdio 模式

Stdio 模式适用于本机运行的本地 MCP 服务器。

| 字段 | 说明 |
|------|------|
| Command | 启动命令（如 `npx`、`python`） |
| Args | 传递给可执行文件的命令行参数 |
| Working Dir | 进程运行目录 |
| Env JSON | 环境变量，JSON 对象格式 |
| Timeout | 进程启动超时时间（秒） |

## 同步模式

同步模式控制 Arkloop 何时从 MCP 服务器获取工具列表。

| 模式 | 行为 |
|------|------|
| 每次对话前刷新 | 每次新对话开始时刷新工具列表 |
| 首次连接时获取 | 首次连接时获取一次，之后缓存 |

服务器工具列表频繁变化时选"每次对话前刷新"。服务器稳定时选"首次连接时获取"以减少启动延迟。

## 发现 / 扫描 / 导入

Arkloop 可以检测系统中已有的 MCP 配置。

- **发现** — 扫描本机已运行的 MCP 服务器
- **扫描** — 搜索常见配置位置（如 Claude Desktop 配置、VS Code 设置）中的 MCP 条目
- **导入** — 直接添加检测到的配置，无需重新输入

设置 → MCP → 扫描 即可开始。

## 添加后

MCP 服务器添加后，其工具立即出现在 Agent 的工具列表中，可在对话中被调用。无需重启。Agent 根据上下文自主决定何时使用。
