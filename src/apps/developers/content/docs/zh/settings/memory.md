记忆设置控制 Arkloop 如何跨对话存储、组织和检索信息。

## 全局开关

主开关控制整个记忆子系统。关闭后不存储也不召回任何记忆。

设置 → Memory → 启用记忆。

## 自动整理

"每轮后自动整理"开关控制 AI 是否在每条回复后自动将对话内容提炼为记忆。关闭时，记忆更新仅在 AI 主动决定写入时发生。

设置 → Memory → 每轮后自动整理。

## Provider 选择

三选一卡片，选择后决定需要配置的内容。

| Provider | 配置项 |
|----------|--------|
| Notebook | 无，立即生效 |
| OpenViking | 提取模型 + 嵌入模型 + 可选 Rerank 模型 |
| Nowledge | Base URL + API Key + Timeout |

选择方式：设置 → Memory → 选择 Provider 卡片。

## OpenViking 配置

选择 OpenViking 后，弹出配置弹窗。

### 模块状态

状态卡显示 Installer Bridge 状态（在线/离线）和 OpenViking 模块状态（运行/停止/未安装）。

- 未安装：显示"安装"按钮
- 已安装但停止：显示"启动"按钮
- 运行中：可进行配置

### 模型配置

模块运行后，表单提供各模型的下拉选择：

- **提取模型** — 决定从对话中存储什么内容
- **嵌入模型** — 将存储内容向量化以供检索
- **Rerank 模型**（可选）— 对候选结果重排序，提升检索精度

点击"保存并配置"应用。

## Nowledge 配置

选择 Nowledge 后，填写：

- **Base URL** — Nowledge 实例的根地址
- **API Key** — 实例的认证凭证
- **Timeout** — 请求超时秒数

"检测本地实例"按钮扫描 localhost，如发现运行中的 Nowledge 服务则自动填充 Base URL。

## 印象画像

印象画像（UI 中标注为 Impression）是 AI 对你的整体理解，跨所有对话累积。

- **查看** — 阅读当前印象内容
- **重建** — 从全部对话历史重新生成印象

设置 → Memory → 印象画像 → 查看或重建。

## 快照

快照（UI 中标注为 Memories）显示当前记忆条目，按深度层级组织。

- **L1** — 精简摘要条目
- **L2** — 含更丰富上下文的详细条目

可浏览各层级条目、查看单项内容，或从原始对话数据重建快照。

设置 → Memory → 快照 → 查看或重建。

## 错误日志

记忆操作遇到失败时，显示提示："检测到近期记忆错误"。点击打开错误日志，列出每次失败操作的时间戳和错误信息。用于排查配置问题或后端连通性故障。
